## 🔗 需要改 2+ 個文件的情況

| 需求 | 要改的文件 | 順序 |
|-----|----------|------|
| 新排版（3x3） | CSS → theme JSON | 先定義再設定 |
| 新關卡 | questions → config → HTML | 先題目再設定再按鈕 |
| 切換主題 | 確認主題資料夾 → HTML | 先準備資料再切換 |

**黃金規則：一次改一個，改完測試！** ✅

## 🚦 改動風險等級

```
🟢 綠燈（放心改）
   - theme-v2.json 的顏色和文字
   - questions-v2.json 的內容
   - config-v2.json 的數值

🟡 黃燈（小心改）
   - style.css
   - index.html 的文字
   - 新增內容

🔴 紅燈（找工程師）
   - game-engine.js
   - theme-loader.js
   - 改邏輯
```

## 🎯 最常見的 5 種改動

### 1. 改顏色
```
文件：theme-v2.json
位置："colors"
風險：🟢 安全
```

### 2. 改題目
```
文件：questions-v2.json
工具：question-editor-v2.html
風險：🟢 安全
```

### 3. 改分數
```
文件：config-v2.json
位置："targetScore"
風險：🟢 安全
```

### 4. 改排版
```
文件：theme-v2.json + style.css
風險：🟡 小心
```

### 5. 改邏輯
```
文件：game-engine.js
風險：🔴 危險
建議：找工程師
```

## 🔍 檢查：這個改動安全嗎？

```
問自己：
□ 只改 JSON 的數字/文字？      → 🟢 安全
□ 要改 CSS 樣式？              → 🟡 小心
□ 要改 HTML 結構？             → 🟡 小心
□ 要改 JavaScript 邏輯？       → 🔴 危險
```
